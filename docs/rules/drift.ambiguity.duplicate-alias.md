# `drift.ambiguity.duplicate-alias`

| field    | value |
|----------|-------|
| category | `ambiguity` |
| default  | `error` |
| fixable  | `no` |

## what

two tables with the same alias in one FROM clause is ambiguous

## bad

```sql
SELECT u.id, o.id
FROM users u
JOIN orders u ON u.user_id = u.id;
```

## good

```sql
SELECT u.id, o.id
FROM users u
JOIN orders o ON o.user_id = u.id;

```
