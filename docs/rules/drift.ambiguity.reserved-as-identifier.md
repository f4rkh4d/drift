# `drift.ambiguity.reserved-as-identifier`

| field    | value |
|----------|-------|
| category | `ambiguity` |
| default  | `warning` |
| fixable  | `no` |

## what

using a reserved word as an identifier forces quoting everywhere it appears

## bad

```sql
CREATE TABLE "order" (id int);
```

## good

```sql
CREATE TABLE orders (id int);

```
