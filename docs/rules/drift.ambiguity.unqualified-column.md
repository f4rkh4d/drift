# `drift.ambiguity.unqualified-column`

| field    | value |
|----------|-------|
| category | `ambiguity` |
| default  | `info` |
| fixable  | `no` |

## what

when two tables are joined, every column reference should be qualified

## bad

```sql
SELECT id, name
FROM users
JOIN profiles ON users.id = profiles.user_id;
```

## good

```sql
SELECT users.id, users.name
FROM users
JOIN profiles ON users.id = profiles.user_id;

```
