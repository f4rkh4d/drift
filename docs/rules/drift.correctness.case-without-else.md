# `drift.correctness.case-without-else`

| field    | value |
|----------|-------|
| category | `correctness` |
| default  | `info` |
| fixable  | `no` |

## what

CASE without ELSE returns NULL for unmatched rows — usually unintended

## bad

```sql
SELECT
  CASE status
    WHEN 'paid'    THEN 1
    WHEN 'pending' THEN 2
  END AS rank
FROM orders;
```

## good

```sql
SELECT
  CASE status
    WHEN 'paid'    THEN 1
    WHEN 'pending' THEN 2
    ELSE 0
  END AS rank
FROM orders;

```
