# `drift.correctness.missing-where-update`

| field    | value |
|----------|-------|
| category | `correctness` |
| default  | `error` |
| fixable  | `no` |

## what

UPDATE without WHERE rewrites every row. almost always a mistake.

## bad

```sql
UPDATE users SET active = 0;
```

## good

```sql
UPDATE users SET active = 0 WHERE last_login < '2024-01-01';

```
