# `drift.performance.fn-on-column`

| field    | value |
|----------|-------|
| category | `performance` |
| default  | `info` |
| fixable  | `no` |

## what

calling a function on a column in WHERE prevents the index from being used

## bad

```sql
SELECT * FROM users WHERE LOWER(email) = 'foo@bar.com';
```

## good

```sql
-- store the lowercased value, or add a functional index:
CREATE INDEX users_email_lower ON users (LOWER(email));
SELECT * FROM users WHERE LOWER(email) = 'foo@bar.com';

```
