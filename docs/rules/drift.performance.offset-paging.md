# `drift.performance.offset-paging`

| field    | value |
|----------|-------|
| category | `performance` |
| default  | `info` |
| fixable  | `no` |

## what

OFFSET is O(n) in most engines; prefer keyset paging for deep pages

## bad

```sql
SELECT * FROM events ORDER BY created_at LIMIT 50 OFFSET 100000;
```

## good

```sql
-- keyset pagination is O(log n):
SELECT * FROM events
WHERE created_at > $last_seen_ts
ORDER BY created_at
LIMIT 50;

```
