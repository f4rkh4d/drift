# `drift.portability.double-quote-ident`

| field    | value |
|----------|-------|
| category | `portability` |
| default  | `info` |
| fixable  | `no` |

## what

double quotes are identifiers in ansi but strings in some mysql configs

## bad

```sql
-- in mysql this means string, not identifier:
SELECT * FROM "users";
```

## good

```sql
-- ANSI: double-quoted is identifier; mysql needs backticks or ANSI_QUOTES on:
SELECT * FROM users;

```
