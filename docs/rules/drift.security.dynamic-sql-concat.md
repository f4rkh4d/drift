# `drift.security.dynamic-sql-concat`

| field    | value |
|----------|-------|
| category | `security` |
| default  | `warning` |
| fixable  | `no` |

## what

concatenated sql in stored procs is an injection smell

## bad

```sql
EXECUTE 'SELECT * FROM users WHERE id = ' || user_input;
```

## good

```sql
EXECUTE 'SELECT * FROM users WHERE id = $1' USING user_input;

```
