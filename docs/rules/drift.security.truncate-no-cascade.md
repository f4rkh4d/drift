# `drift.security.truncate-no-cascade`

| field    | value |
|----------|-------|
| category | `security` |
| default  | `info` |
| fixable  | `no` |

## what

TRUNCATE semantics differ across engines; state cascade/restrict explicitly

## bad

```sql
TRUNCATE orders;
```

## good

```sql
-- if FKs depend on it, be explicit:
TRUNCATE orders RESTRICT;
-- or, when you actually want the cascade:
TRUNCATE orders CASCADE;

```
