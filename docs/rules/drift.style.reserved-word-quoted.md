# `drift.style.reserved-word-quoted`

| field    | value |
|----------|-------|
| category | `style` |
| default  | `info` |
| fixable  | `no` |

## what

avoid using reserved keywords as identifiers, even quoted

## bad

```sql
SELECT "order".id FROM "order";
```

## good

```sql
SELECT orders.id FROM orders;

```
